# Project Protection Rules
**CRITICAL INSTRUCTIONS FOR AI AGENT:**

1. **NEVER DELETE EXISTING FEATURES:** Do not remove, comment out, or overwrite existing UI elements, configurations, features, or logic unless the user explicitly requests their deletion.
2. **PRESERVE ALL CODE:** When modifying a file to add a new feature, you must keep all previous variables, states, imports, and functions intact.
3. **SURGICAL EDITS ONLY:** Use accurate, minimal replacements when editing code. Do not rewrite entire components or files if you only need to add a few lines.
4. **NO FEATURE REGRESSION:** Always verify that your changes do not break or hide any previously built features.

These rules are permanent to ensure that as the app grows, no existing work is accidentally lost.
